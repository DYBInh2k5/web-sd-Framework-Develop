============== THỰC HÀNH TUẦN 8 - BLADE - SỬ DỤNG COMPONENT ===========
1/ Tạo component Alert bằng lệnh 
	php artisan make:component Alert
	
	Sau khi tạo thành công sẽ phát sinh 2 file mới
	 - app/View/Components/Alert.php // đóng vai trò là controller
	 - resources/views/components/Alert.Blade.php //vai trò là view
2/ - Mặc định các bạn muốn dùng AlertComponent thì chỉ cần khai báo trên view : <x-alert/>

   - Tuy nhiên nếu các bạn muốn đặt tên lại và sử dụng với tên thẻ khác so với mặc thì các bạn cần 
     đăng kí component với tên mới trước khi sử dụng trong file : app/Providers/AppServiceProvider.php như sau:
	
	use Illuminate\Support\Facades\Blade;
	use App\View\Components\Alert;
	...
	public function boot()
    {
        Blade::component('package-alert', Alert::class);
    }
3/ Viết sử lý cho component
	- Viết cho file app/View/Components/Alert.php như sau:
		class Alert extends Component
		{	
			public $message;
			public $type;

			public function __construct($type, $message)//khi khai báo sử dụng component thì phải truyền giá trị cho type và message 
			//vì đây là hàm khởi tạo
			{
				$this->type = $type;
				$this->message = $message;
			}		
			public function render()
			{
				return view('components.alert'); // render tới view alert nằm trong thư mục components
			}
		}
	- Viết xử lý cho file resources/views/components/Alert.Blade.php (phần giao diện của component)
		<div class="alert alert-{{ $type }}">
			{{ $message }}
		</div>
4/ Gọi sử dụng component: hãy chọn 1 view mà bạn muốn nhúng component Alert vào và khai báo như sau:
	...
	<x-package-alert type="danger"  message="demo component"/> 
	...
	Nếu giá trị các bạn truyền cho component không phải các giá trị của kiểu dữ liệu nguyên thủy (string, integer,..)
	mà là các biến nhận từ view cha thì các bạn khai báo như sau:
	...
	<x-package-alert type="danger"  :message="$msgFromParent"/> 
	...
5/ - Bổ sung code cho alert component sao cho component này có thể hỗ trợ thông báo cho 1 message hay danh sách các message 
	(dùng trong trường hợp 1 thông báo có chứa nhiều thông báo lỗi cùng 1 lúc)
   - Lưu ý component này sẽ được sử dụng cho các thông báo lỗi trên các trang (có tính vào yêu cầu của project)
